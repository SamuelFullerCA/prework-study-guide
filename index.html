HTML

  • Hypertext Markup Language HTML adds structure and framwork to a webpage.
  • HTML is made up of elements that work together such as text, images, headers, and lists.
  • Elements can also divide the content on a page and describe it to browsers and developers.
  • These elements are: Meta-related, semantic and Content division.
  • The head element contains information or metadata that helps the browser to render the page correctly.
  • The body element represents the visible content shown to the user.

    • Head Element

  • !DOCTYPE tells the browser what the document type it should expect.
  • The lang="en" element is an attribute that tells the browser language of the page.
  • The title element defines the title for a webpage. Extremly important for Search Engine Optimization.
  • The style element defines the CSS styles associated with the website.
  • The base element defines the base URL for a webpage.
  • The link elment connects an external resource to the HTML document.
  • The meta element defines metadata such as character set, description keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The charset element sets the character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for out webpage.

    • Body Element

  • Body uses semantic HTML which means the name of elements describes its content and role.
  • Not all elements are semantic suc as div, which creates a division in the page, or span which creates an inline division.
  • The h# element makes a header with the number repeesenting hte level of heading in a given text block.
  • The p element represents a paragraph or block of text.
  • The ul tag creates and unordered list of items.
  • The ol tage creates an ordered list of items.
  • The li tag sits inside the ul to wrap each list item as a bullet point.
  • The id element is used by HTML as an attribute to identify elements and select them.
  • The img element contains information about images that are displayed.
  • The img element can contain various attributes with src being the most important as it defines the location of the image.
  • The br element creates a line of empty space or a line break between two blocks of content.
  • The a element or anchor creates a link to the same webpage or other webpages.
  • The section elements are semantic elemets that define a section in a document that contains thematic content.

CSS

  • Cascading Style Sheets, CSS, is used to define how a webpage contant is presented to users such as style and color.
  • Three methods of CSS are inline, internal style sheet, and external style sheet.
  • Create and external file with style.css in assents folder.
  • Link the file with the link element placed in the HTML's head.
  • The rel attribute specifies the relationship between the current document and linked document.
  • The href attribute specifies the location of the external resource.
  • CSS Rule - Div {color: blue; text-align: justify;}.
  • CSS Rule - Selector {declaration; Declaration}.
  • Declaration contains two important components. The CSS property and the value of the property.
  • A margin property indicates how much space we want around the outside of a element.
  • The padding property adds space around the inside of an element.
  • The border property assigns a border with a width of 5px that is solid grey.
  • The box-shadow property adds a shadow effect around an element's frame.
  • A box shadow is described by X and Y offsets relative to the element, blur and spread radius, and color.
  • Hexidecimal code can be used to define color.
  • CSS rules can also be applied to headers, footers, lists, p texts, and images.
  • Two or more elements that will share the same styling can be grouped together.
  • Write DRY code, do not repeat yourself.

Git

  • Creating a new branch:
  • 1. Navagate to correct directory.
  • 2. Check what branch currently in by running git status.
  • 3. If not in main branch, run git checkout main to switch.
  • 4. Pull github's main branch to local main by running git pull origin main.
  • 5. Create new branch with git checkout -b branchname.
  • 6. Type code to code!
  • Pushing branch with commit to github.
  • 1. Stage changes with git add -A.
  • 2. Add commit message with git commit -m message.
  • 3. Make sure local branch is up to date with git pull origin main.
  • 4. Compleate the flow with git push origin branchname.

JavaScript

  • A variable is a named container that allows is to store data in our code.
  • Output messages to the web console by adding an argument. An example would be console.log().
  • Linking to main HTML page similar to CSS stylesheet but with script element.
  • Access Chrome DevTools with right click - inspect. Or Command+Option+I.
  • Control flow is the order in which a computer excudes code in a script.
  • JavaScript code runs form the first statement at the top of the page, down to the last statement at the bottom of the page.
  • Control flow can be interrupted with conditional statements and loops.
  • An if statement is a way of introducing decision-making into code. The code will only execute if the statement meets a condition or is true.
  • JavaScript will check to see if if statemsnts are truthy. Truthy statements are: 0, -0, On, "", null, undefined, NaN, or a Boolean.
  • The if statement can have more than one codition by using else if.
  • Computer will continue through all conditions in a if else list intil it meets a condition that is truthy, then execude the code in yhe curly brackets assocated with the statement.
  • The === is a strict equality operator.
  • Strings such as names need to be contained within quotations marks.
  • An Array is a list of multiple data types in a single variable.
  • Data in an array is given an index or unique number starting with 0 and increasing by 1.
  • A +1 loop shortcut is x++.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • A function must be called in the code to be executed.
